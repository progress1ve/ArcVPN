# Инструкция по настройке ArcVPN

## Что было изменено:

### 1. ✅ Проверка подписки на канал
- При входе в бота пользователь должен подписаться на канал @arcvpn1
- Админы пропускаются без проверки

**Настройка:**
Откройте файл `bot/middlewares/subscription_check.py` и измените:
```python
REQUIRED_CHANNEL_ID = "@arcvpn1"  # Ваш канал
REQUIRED_CHANNEL_LINK = "https://t.me/arcvpn1"  # Ссылка на канал
```

### 2. ✅ Обновлено главное меню
- Показывается имя пользователя, ID и баланс
- Новостной канал: @arcvpn1
- Техническая поддержка: @progressive_dev

**Изменить контакты:**
В файле `bot/handlers/user/start.py` в функции `get_welcome_text()` измените:
```python
f"<b>Новостной канал</b> — @arcvpn1\n"
f"<b>Техническая поддержка</b> — @progressive_dev"
```

### 3. ✅ Переименованы кнопки
- "Мои ключи" → "📱 Мои подписки"
- "Купить ключ" → "💳 Купить подписку"
- "Реферальная ссылка" → "🤝 Партнерская программа"
- "Справка" → "ℹ️ О сервисе"

### 4. ✅ Новое расположение кнопок
Теперь кнопки расположены вертикально:
1. Мои подписки
2. Купить подписку
3. Партнерская программа
4. О сервисе
5. Пробная подписка (если доступна)
6. Админ-панель (для админов)

### 5. ⚠️ Реферальная система (требует доработки)
Текущая система уже поддерживает начисление на баланс.

**Чтобы изменить на фиксированные 100₽:**
Нужно изменить логику в файле `bot/services/billing.py` в функции начисления реферальных вознаграждений.

## Запуск бота:

```bash
python main.py
```

Или через systemd:
```bash
systemctl restart arcvpn
```

## Важно:

1. **Добавьте бота в администраторы канала** @arcvpn1 чтобы он мог проверять подписку
2. Убедитесь что канал публичный или бот имеет права на чтение участников
3. При первом запуске применится миграция v16 (добавление поля first_name)

## Тестирование:

1. Запустите бота
2. Отправьте /start (не будучи подписанным на канал)
3. Должно появиться сообщение о необходимости подписки
4. Подпишитесь на канал
5. Нажмите "Я подписался"
6. Должно открыться главное меню с вашим именем и балансом
